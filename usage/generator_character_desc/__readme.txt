--##############
--#### ПОЛЯ ####
--##############
name
    Имя НПС.
    Если не указано, то восстанавливается автоматически по группировке и, в случае военных, по рангу.
icon
    Иконка.
    Если не указана, то
        - для группировки зомбированных приравнивается ui_npc_u_none
        - в остальном, формируется по принципу ui_npc_u_<visual_name>
class
    Класс, т.е. имя характеристики.
    Указывается ОБЯЗАТЕЛЬНО.
community
    Группировка НПС.
    Указывается ОБЯЗАТЕЛЬНО.
terrain_sect
    Секция террейна, что бы то ни значило.
    Если не указано, то просто не добавляется в характеристику.
rank
    Ранг НПС.
    Если указано число, то таким будет ранг НПС.
    Если указаны два числа через запятую, то ранг для каждой характеристики выбирается случайно между этими числами.
    Можно указать сокращения, обозначающие опытность НПС, и число ранга будет выбрано случайным образом в рамках указанной опытности. Возможные сокращения: novice, experienced, veteran, master, 1n, 2e, 3v, 4m.
    Указывается ОБЯЗАТЕЛЬНО.
reputation
    Репутация НПС.
    Если указано число, то такой будет репутация НПС.
    Если указаны два числа через запятую, то репутация для каждой характеристики выбирается случайно между этими числами.
    Указывается ОБЯЗАТЕЛЬНО.
money
    Кол-во денег НПС.
    Если указано одно положительное число, то min и max ему приравниваются.
    Если указаны два положительных числа через запятую, то они соответствуют min и max.
    Если хотя бы одно число (из одного или двух) отрицательное, то устанавливается бесконечное кол-во денег.
    Указывается ОБЯЗАТЕЛЬНО.
    ВНИМАНИЕ. Не указывай нулевые значения! Видимо движок интерпретирует это как отсутствие money в характеристике, так что этот параметр будет браться из all.spawn.
visual
    Модель НПС.
    Указывается ОБЯЗАТЕЛЬНО.
snd_config
    Звуковая схема НПС.
    Указывается ОБЯЗАТЕЛЬНО.
crouch_type
    Не знаю, что это такое.
    В оригинале принимает 3 значения: -1, 0, 1. Есть нестрогая корреляция этого значения с группировкой НПС.
    Лучше не указывать. В этом случае скрипт подберёт одно и трёх значения по группировке НПС.
w0, w1, w2
    Секции оружия, указываются через запятую.
    В зависимости от выбранного режима генерации (_genmode) указанное оружие будет скомбинировано друг с другом.
    Смысл полей такой:
        w0 - одиночное оружие
        w1 - оружие первого слота (пистолеты)
        w2 - оружие второго слота (автоматы, дробовики, снайперки, гранатомёты)
    Можно указать пустую строчку. В таком случае генерация будет воспринимать это как отдельную специальную единицу секции оружия - "без оружия".
    Если требуется сбросить список оружия (например, при наследовании секций настроек), то нужно в значение поля прописать None.
    В скобках после секции можно указать дополнительные параметры (синтаксис - как в [spawn]), например:
        w2 = wpn_pm (1, cond=0.8 scope silencer), wpn_pb (1, scope), wpn_raging_bull
    Если в дополнительных параметрах указаны нулевое количество или нулевая вероятность, то этот предмет будет считаться "пустым" (т.е. этот предмет будет учитываться при выборе комбинации оружия, но в характеристике прописываться не будет); иными словами, следующие строки равносильны:
        w0 = wpn_bizon, wpn_ump45 (0), wpn_vihr
        w0 = wpn_bizon, wpn_ump45 (1, prob=0), wpn_vihr
        w0 = wpn_bizon,, wpn_vihr
items
    Дополнительные предметы для НПС.
    Перечисляются через запятую.
    Если требуется сбросить список предметов (например, при наследовании секций настроек), то можно в значение поля прописать None на манер того, как это делается с оружием (w0, w1, w2), но в случае поля items можно в значении оставить просто пустоту.
    В скобках после секции можно указать дополнительные параметры (синтаксис - как в [spawn]), например:
        items = bread (3), kolbasa (2), vodka, device_torch (1, prob=0.5)
    Если в дополнительных параметрах указаны нулевое количество или нулевая вероятность, то этот предмет прописываться в характеристике не будет; иными словами, следующие строки равносильны:
        items = bread, kolbasa (0), conserva
        items = bread, kolbasa (1, prob=0), conserva
        items = bread,, conserva
include_supplies
    Включение файлов с доп. предметами для НПС, которые будут у него, пока он жив.
    Имена файлов перечисляются через запятую. Возможные варианты:
    character_drugs, character_food, character_items
    По умолчанию ничего не добавляется.
include
    Включение файлов с диалогами и critical_wound_weights.
    Значение по умолчанию: character_criticals, character_dialogs
    Не рекомендую пропускать character_criticals (не знаю точно, зачем тамошний critical_wound_weights, но я его всегда указывал).
    [vanilla] Файла character_criticals из значения по умолчанию в оригинальных ресурсах игры нет, поэтому нужно либо его добавить, либо в открытую указать какой-нибудь из существующих. Какой из character_criticals_* включать именно не имеет значения - они все одинаковые (можно указать, например, character_criticals_3).
_genmode
    Принцип генерации по указанной настройке характеристик (без кавычек):
    "0" или "w0+w1*w2"  | Будет сгенерировано len(w0) + len(w1)*len(w2) характеристик.
                          Т.е. для каждого автомата будет перебран каждый пистолет.
                          Значение по умолчанию.
    "1" или "unique"    | Уникальная характеристика: len(w0) + len(w1)*len(w2) == 1.
                          Для id характеристики в xml не будет добавлен постфикс _defaultN.
    "2" или "w0+w2"     | Будет сгенерировано len(w0) + len(w2) характеристик.
                          Для каждого автомата будет выбран детерминированно один пистолет.
                          В каждой новой настройке с тем же классом индекс для выбора пистолета смещается,
                          так что одному автомату не будет всегда соответствовать один и тот же пистолет.


--########################
--#### ПРИНЦИП РАБОТЫ ####
--########################
Каждая секция, не начинающаяся с нижнего подчёркивания, - настройка характеристик.
Секции, начинающиеся с нижнего подчёркивания, - абстрактные, т.е. не участвуют в генерации.
Абстрактные секции удобны для наследования полей секции.
Для каждой настройки будет сгенерирован ряд характеристик с отличающимся вооружением.
Принцип генерации задаётся настройкой _genmode.
В общем случае, w0 - оружие, выдающееся индивидуально, а w1 и w2 комбинируются между собой.
Патроны для оружия подбираются автоматически.
Реализована поддержка многоприцельности: если в секции оружия существует поле scope_respawn и scope_status = 2, то прицел будет прикреплён автоматически.
